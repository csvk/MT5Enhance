
FOREX PNL EXAMPLE CALCULATIONS (USD ACCOUNT)
==========================================

This document provides step-by-step PnL calculation examples for Forex trades
covering USD-quoted pairs, USD-base pairs, and non-USD cross pairs.
All examples assume a USD-denominated trading account.

--------------------------------------------------
GENERAL DEFINITIONS
--------------------------------------------------
- Standard lot size = 100,000 units of base currency
- Pip size:
  * Non-JPY pairs: 0.0001
  * JPY pairs:     0.01
- BUY  = price increase is profit
- SELL = price decrease is profit

--------------------------------------------------
EXAMPLE 1: EURUSD (QUOTE CURRENCY = USD)
--------------------------------------------------
Pair:      EURUSD
Direction: BUY
Entry:     1.1000
Exit:      1.1025
Lots:      0.10

Price move:
1.1025 - 1.1000 = 0.0025

Pips:
0.0025 / 0.0001 = 25 pips

Pip value per lot:
100,000 × 0.0001 = $10

PnL:
25 × (10 × 0.10) = $25.00

--------------------------------------------------
EXAMPLE 2: USDJPY (BASE CURRENCY = USD)
--------------------------------------------------
Pair:      USDJPY
Direction: SELL
Entry:     92.377
Exit:      91.871
Lots:      0.05

Price move:
92.377 - 91.871 = 0.506

Pips:
0.506 / 0.01 = 50.6 pips

Pip value:
100,000 × 0.01 = 1,000 JPY
Converted to USD:
1,000 / 91.871 ≈ $10.88 per pip per lot

PnL:
50.6 × (10.88 × 0.05) ≈ $27.54

--------------------------------------------------
EXAMPLE 3: EURGBP (NON-USD CROSS PAIR)
--------------------------------------------------
Pair:      EURGBP
Direction: BUY
Entry:     0.8500
Exit:      0.8520
Lots:      1.00
GBPUSD:    1.25

Price move:
0.8520 - 0.8500 = 0.0020

Pips:
0.0020 / 0.0001 = 20 pips

PnL in GBP:
20 × (100,000 × 0.0001) = 200 GBP

PnL in USD:
200 × 1.25 = $250.00

--------------------------------------------------
KEY TAKEAWAYS
--------------------------------------------------
- USD-quoted pairs need no FX conversion
- USD-base pairs require division by price
- Cross pairs require conversion using a USD pair
- Point-based calculations avoid pip-size ambiguity
